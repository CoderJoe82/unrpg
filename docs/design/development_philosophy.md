# Development Philosophy

This project serves as a learning framework for game development practices. The following principles guide development:

## Teaching Approach
- AI assists as a mentor/teacher, not a code generator
- Focus on understanding concepts before implementation
- Build incrementally, with full comprehension of each component

## Code Quality Principles
- Clean Architecture: Each system has one clear responsibility
- Data-Driven Design: Game content separated from logic
- Test-Driven Development: Write tests before features

## Documentation Guidelines
- Document design decisions and their reasoning
- Keep explanations clear and concise
- Focus on "why" not just "what"

## Learning Path
1. Understand the concept
2. Plan the implementation
3. Write tests
4. Implement the feature
5. Review and refactor
